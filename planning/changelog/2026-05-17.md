### [#840 feat: stabilize system prompt prefix caching (#763)](https://github.com/allenhutchison/obsidian-gemini/pull/840)

Resolves the quadratic prompt parsing costs described in #763 by stabilizing Gemini's prefix cache across agent turns. Per-turn context (context-chip files, attachment metadata) is no longer embedded in `systemInstruction` — instead it's spliced directly into the user message parts for each turn. The result is a completely static `systemInstruction` throughout the session, giving a 100% prefix cache hit rate and roughly 75% lower input token cost on long conversations. `buildToolHistoryTurns` in `agent-loop-helpers.ts` was updated to propagate `perTurnContext` through tool follow-up iterations so file context remains visible to the model across multi-step tool calls. Tests updated across `agent-loop-helpers.test.ts`, `gemini-prompts.test.ts`, and the provider client tests.

### [#843 fix: resolve install:test-vault target by manifest id](https://github.com/allenhutchison/obsidian-gemini/pull/843)

`npm run install:test-vault` hardcoded the destination folder name (`gemini-scribe`), so a test vault whose plugin folder was named anything else (e.g. `obsidian-gemini`) was silently missed — the script refreshed a folder Obsidian never loaded. The fix scans `<vault>/.obsidian/plugins/*/manifest.json` for the plugin `id` and installs into whichever folder declares it. `docs/contributing/testing.md` updated to match. This bug was surfaced during QA of #840, where the first smoke-test run silently exercised a stale build.
