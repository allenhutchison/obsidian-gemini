## What shipped

### [#909 feat: configurable headless tool-iteration cap for tasks and hooks](https://github.com/allenhutchison/obsidian-gemini/pull/909)

Scheduled tasks and lifecycle hooks previously capped the headless agent loop at a hardcoded 20 tool-call iterations with no way to raise it. Long-running multi-step tasks would fail with an `exhausted 20 tool iterations` error and the only workaround was to restructure the task. This PR makes the cap configurable via a `maxIterations` frontmatter key on both task and hook files, with a UI field under _Advanced options_ in the Scheduler and Hook editor modals (placeholder shows the default of 20). The interactive agent chat remains uncapped. As part of the refactor, the two previously-duplicated hardcoded `20` constants and the duplicated frontmatter parser were consolidated into single shared definitions (`DEFAULT_HEADLESS_MAX_ITERATIONS` in `agent-loop.ts`, `parseMaxIterations()` in `feature-definition.ts`). Docs and tests were included in the PR.
