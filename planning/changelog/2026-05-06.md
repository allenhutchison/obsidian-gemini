### [#764 feat(settings): collapsible sections (closes #430 foundation)](https://github.com/allenhutchison/obsidian-gemini/pull/764)

Reorganises the settings tab into 13 collapsible `<details>` sections so users can scan headings instead of scrolling past every control. Expand state is persisted in `expandedSettingsSections` so the layout survives plugin reloads. Fixes a pre-existing bug where changing the AI provider would tear down later-rendered sections (UI / Context / Advanced) by re-rendering the full container; now only the General section re-renders. ### [#766 feat(evals): baseline-compare workflow with auto-compare and bless](https://github.com/allenhutchison/obsidian-gemini/pull/766)

Closes #717. After every `npm run eval`, the harness now automatically compares the result against the blessed baseline for the active `(provider, model)` pair and prints a regression-only summary — no more manual `eval:compare` + eyeballing. Operators promote a result to baseline with `npm run eval:bless`; baselines never auto-drift. Regression flagging is conservative: aggregate `pass^k`/`solve^k` drops are flagged, as are per-task solved/passed fraction drops. Cross-provider comparisons mark cost/cache aggregates as not-applicable. Adds `evals/baselines/.gitkeep` so the directory exists before any bless operation, and 16 new tests covering the comparison and baseline-resolution logic.

### [#767 feat(evals): richer rubric matchers + LLM-as-judge fallback](https://github.com/allenhutchison/obsidian-gemini/pull/767)

Closes #713. Two additions that make eval solve rate measure "the agent did the wrong thing" rather than "the agent phrased it differently." First, `contains` and `regex` matchers now accept an array `value` (logical OR across forms), catching wikilink-vs-title-case mismatches without inflating pass criteria. Second, a new `judge` matcher type (`{ "type": "judge", "criteria": "..." }`) calls a pinned Gemini judge model at `temperature: 0` for a strict YES/NO verdict — the judge always uses Gemini even when the system under test is Ollama, so verdicts stay stable across model-swap experiments. Updates `find-tagged-notes` to accept title-case or wikilink slug for each expected note, and replaces `multi-file-summary`'s brittle substring checks with a single `judge` matcher. Eighteen new unit tests cover the full matcher engine.
