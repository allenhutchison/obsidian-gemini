### [#756 feat(hooks): lifecycle event hook engine — foundation (#749 PR 1)](https://github.com/allenhutchison/obsidian-gemini/pull/756)

Foundation of the lifecycle-hooks system planned in #749. Hooks let users register a headless agent task that fires in response to vault events — `file-created`, `file-modified`, `file-deleted`, `file-renamed` — with debounce, rate limit, cooldown, and loop-prevention guards.

Key additions: `src/services/hook-manager.ts` (event subscription, glob/frontmatter filter, debounce queue, auto-pause on hard loop ceiling or consecutive failures), `src/services/hook-runner.ts` (headless execution mirroring ScheduledTaskRunner), `hooksEnabled` setting (default `false`), and a full `docs/guide/lifecycle-hooks.md`. The engine is gated off by default — no folders are created, no listeners registered, until the user opts in. 27 new unit tests. The UI and BackgroundTaskManager wiring land in PR 2.

### [#757 feat(hooks): management UI, BackgroundTaskManager wiring, settings toggle (#749 PR 2)](https://github.com/allenhutchison/obsidian-gemini/pull/757)

Phase 2 of the lifecycle-hooks epic. Hooks are now authorable through a CRUD modal (`src/ui/hook-management-modal.ts`) instead of requiring hand-edited markdown files, and hook fires submit through `BackgroundTaskManager` so they surface in the Activity modal alongside scheduled tasks and deep research.

New settings section (General → Lifecycle Hooks) with the enable/disable toggle and Open/New buttons. Two new command palette entries: `Gemini Scribe: Open Hook Manager` and `Gemini Scribe: New Lifecycle Hook`. Toggling `hooksEnabled` re-initialises the manager directly without a full plugin reload. Docs updated for the modal-first workflow; sidebar entry added. 16 new tests (CRUD, cancellation propagation, bg-manager submission fallback). Suite at 1585 passing.
