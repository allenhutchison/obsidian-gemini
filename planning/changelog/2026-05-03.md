### [#741 feat(scheduled-tasks): add time-of-day and day-of-week schedule formats](https://github.com/allenhutchison/obsidian-gemini/pull/741)

Closes #727. Adds two new schedule string forms to the scheduler: `daily@HH:MM` (run every day at a fixed local time) and `weekly@HH:MM:DAYS` (run at a fixed local time on specified weekdays, e.g. `weekly@16:30:mon,tue,wed,thu,fri`). The existing `daily` and `weekly` bare formats are fully backward-compatible; they gain clearer labels ("Daily (every 24h)" / "Weekly (every 7d)") in the modal to distinguish them from the new time-anchored variants. The scheduler UI gains an HTML5 `<input type="time">` and a weekday checkbox row that compose the schedule string on save and pre-fill when editing an existing time-based task. Twenty new tests cover happy paths, same-tick edge cases, end-of-week wrap, and validation errors.

### [#742 feat(skills): add plugin-test skill and refresh obsidian-cli skill](https://github.com/allenhutchison/obsidian-gemini/pull/742)

Adds the `plugin-test` skill — a deterministic three-pass acceptance workflow (smoke checks, UI/state via Obsidian CLI, optional API-spending pass) that produces a structured report under `planning/test-reports/<timestamp>/`. The skill includes a mandatory vault-guard preflight because the Obsidian CLI's `vault=` flag silently targets the focused window regardless of the name passed. Alongside this, the `obsidian-cli` skill is refreshed from 1.0 to 1.1 with significantly expanded coverage: UI driving via CDP, mobile emulation, frontmatter properties, hotkeys, app lifecycle, file CRUD, and a "Footguns" section documenting 10 traps (including the silent `vault=` fallback).

### [#743 fix: settings render race + ai-writing doc-label drift](https://github.com/allenhutchison/obsidian-gemini/pull/743)

Follow-ups from the 2026-05-03 acceptance test run. Guards `ObsidianGeminiSettingTab.display()` against concurrent invocations using a `renderToken` pattern: each call captures a token and bails out after any `await` if a newer call has superseded it. Without this, chained calls to open settings (e.g. from the API) appended duplicate copies of every settings section. A regression test at `test/ui/settings-display-race.test.ts` uses a deferred-promise gate to interleave a second `display()` call mid-render and asserts each section renders exactly once. Also fixes two stale command-label references in `docs/guide/ai-writing.md` (command palette: "Rewrite text with AI"; context menu: "Gemini Scribe: Rewrite Text..."), verified against the actual `addCommand` calls in `src/main.ts`. The test report from the acceptance run is bundled under `planning/test-reports/2026-05-03-12-18/`.
