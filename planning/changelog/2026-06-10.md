## What shipped

### [#965 feat(agent): persist and display model reasoning (thoughts)](https://github.com/allenhutchison/obsidian-gemini/pull/965)

Fixes #428. Reasoning from thinking models (Gemini 2.5 Flash/Pro with `thinkingBudget` enabled) was previously shown in the progress bar during a turn but thrown away when it completed. This PR captures it end-to-end: stores it on `GeminiConversationEntry.thoughts`, serializes it as a collapsed `> [!reasoning]-` callout in the markdown session history, and re-renders it from saved files when a session is reopened.

The implementation goes all-in: every reasoning step is captured, including the "why I'm calling these tools" thinking that precedes a tool batch (persisted as reasoning-only turns with no message callout). It is always on — no setting. The history parser was updated to key entries on their callouts rather than `##` headers, which keeps reasoning-only turns, multi-callout sequences, and legacy files (no reasoning callout) all round-tripping cleanly. The UI renders collapsible Reasoning sections in the agent view, collapsed by default, interleaved with tool-call groups and the final answer. 9 new tests cover the round-trip, legacy back-compat, and AgentLoop hook behavior.

### [#967 docs(agents): document reasoning persistence, onModelReasoning hook, and typecheck:test](https://github.com/allenhutchison/obsidian-gemini/pull/967)

Docs-only follow-up to #965. Records three non-obvious facts in `AGENTS.md` so future contributors don't rediscover them: (1) `npm run typecheck:test` is required alongside `npm run build` before pushing — CI's lint job type-checks `test/` separately and a green local build does not guarantee green CI (the sole CI failure on #965 was an expression-bodied arrow returning `number` instead of `void`); (2) the `onModelReasoning` hook on `AgentLoopHooks` fires for intermediate pre-tool reasoning, while terminal reasoning comes back on `AgentLoopResult.thoughts`; (3) the session-history parser invariant — entries are identified by callout type, multiple callouts per `---`-delimited section flow together, and old-format fixture guidance for `session-history.test.ts`.
