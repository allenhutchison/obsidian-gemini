### [#949 fix(tools): use canonical resolver in extended vault tools](https://github.com/allenhutchison/obsidian-gemini/pull/949)

Closes #910. Security fix: `UpdateFrontmatterTool`, `AppendContentTool`, and the `append_content` diff-preview branch in `ToolExecutionEngine` each duplicated a 3-step path resolver (direct → `.md` suffix → wikilink) and ran `shouldExcludePath()` only against the raw input string. A bare name like `"Foo"` could resolve via `metadataCache.getFirstLinkpathDest()` to a path inside the plugin state folder, bypassing system-folder protection on both write tools. Replaced with `resolvePathToFile()` from `src/tools/vault/utils.ts`, which applies `shouldExcludePath()` against the resolved path at every resolution strategy. Regression tests added for the wikilink-bypass shape on both write tools and the diff-preview branch.
