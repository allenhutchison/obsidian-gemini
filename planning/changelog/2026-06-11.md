### [#970 refactor(agent): type session listing with TFolder instead of any cast](https://github.com/allenhutchison/obsidian-gemini/pull/970)

The nightly architecture-audit sweep flagged an improper `any` cast in `getAllAgentSessions` (`src/agent/session-history.ts`): the code was checking `'children' in folder` and then accessing `(folder as any).children` rather than using the Obsidian-native `instanceof TFolder` narrowing. Switching to `instanceof TFolder` eliminates the cast and the redundant `any` annotations on the filter/sort callbacks — TypeScript infers the types correctly from `TFolder.children: TAbstractFile[]`. Behaviorally identical; test fixtures updated to construct folders via `new TFolder()` so the `instanceof` check holds in tests.

### [#972 feat(i18n): AI-bootstrapped UI translations with agent panel pilot](https://github.com/allenhutchison/obsidian-gemini/pull/972)

First PR in the #754 localization track. Ships the full i18n framework and 20 committed languages, with the agent panel empty state as the pilot migration.

The framework centers on two files: `src/i18n/en.ts` (the source of truth — 17 keys, each with a translation context string fed to the generation prompt) and `src/i18n/index.ts` (the `t(key, vars?)` helper with `{name}` interpolation and an exact → base-language → English locale fallback). Locale detection uses Obsidian's `getLanguage()` API per the repo's "Obsidian API First" convention — a deliberate departure from the localStorage approach sketched in the issue.

The generator (`scripts/translate.mjs` / `npm run translate`) calls Gemini with structured JSON output to translate stale keys per language. A hash manifest (`src/i18n/translation-state.json`) tracks the English source each translation came from, so re-runs retranslate only changed keys — hand-refined translations survive regeneration. The script never runs in build or CI; the 20 generated files are committed. Languages shipped: cs, da, de, es, fr, id, it, ja, ko, nl, no, pl, pt, pt-BR, ru, tr, uk, vi, zh, zh-TW (aligning with Obsidian's shipped locales; ar/he deferred pending RTL layout work).

The agent panel empty state is fully migrated to `t()`, and non-English locales show an AI-translation disclosure footer. Tests in `test/i18n/` cover fallback chains, interpolation, regional variants, and a registry/file parity check that fails if a generated language file is not registered. Docs updated in README, `docs/guide/faq.md` (localization section + `getLanguage()` fix), and AGENTS.md (command + conventions).
