### [#860 fix: send pure BaseModelRequest from VaultAnalyzer](https://github.com/allenhutchison/obsidian-gemini/pull/860)

Fixes the broken "Initialize Vault Context" button in the agent view. Clicking it produced a JSON parse error (`Unexpected token 'H', "Hi Allen, "... is not valid JSON`) and never wrote `AGENTS.md`. Root cause: `VaultAnalyzer` was passing `userMessage: ''` and `conversationHistory: []` alongside its real `prompt`, which triggered the `'userMessage' in request` discriminator in `GeminiClient.buildGenerateContentParams()` to treat the call as an `ExtendedModelRequest` — discarding the `prompt` entirely and sending the chat identity system prompt with empty content. The model replied with a greeting that failed JSON parsing. Fix removes the stray fields so the request is a pure `BaseModelRequest` matching how `context-manager.ts` calls the same API. A regression test asserts the request carries no `userMessage`/`conversationHistory`. The underlying `'userMessage' in request` discriminator fragility is tracked separately in #859.

### [#862 docs: fix dead link to Testing Guide in contributing.md](https://github.com/allenhutchison/obsidian-gemini/pull/862)

Unblocks the failing "Deploy Documentation" CI workflow. `docs/contributing/contributing.md` linked to the Testing Guide as `docs/contributing/testing.md` — a repo-root-style path — but VitePress resolves markdown links relative to the file's own directory, causing a dead link. The target file lives in the same folder, so the link is corrected to `testing.md`. The broken link was introduced nine days earlier in #760.
