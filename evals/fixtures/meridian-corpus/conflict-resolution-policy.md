# Conflict Resolution Policy

This policy governs how Meridian staff resolve disagreements between documents.

## Recency Rule

When two notes state different values for the same fact, and there is no other guidance, use the value from the note with the more recent `updated` date in its frontmatter. The older note is considered superseded.

## Authority Rule

When a formal specification note disagrees with an informal source such as a chat log, email, or meeting aside, the specification note is authoritative. Use the specification's value regardless of dates.

## Precedence

If the recency rule and the authority rule would give different answers, the authority rule wins: a specification always overrides an informal note.
